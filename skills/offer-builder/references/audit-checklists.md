# Offer Audit Checklists

## Purpose
Steps 10-12 of the offer-building process. Three sequential audit passes that catch different types of problems. Run all three before deploying.

---

## Audit Pass 1: The Stranger Test (Step 10)

**Question:** "Could a stranger who knows nothing about you understand this offer and want it?"

Read the offer document as if you've never heard of this person or product. Check:

### Clarity Checks
- [ ] Can you explain what they sell in one sentence?
- [ ] Is the target buyer obvious (not "everyone")?
- [ ] Is the price or pricing model stated clearly?
- [ ] Are deliverables specific (not "we'll work together")?
- [ ] Is the timeline stated?
- [ ] Is the outcome measurable or at least vivid?

### Jargon Checks
- [ ] No insider terminology without explanation
- [ ] No acronyms that aren't defined
- [ ] No assumed context ("as you know...")
- [ ] Could a smart 16-year-old understand the offer?

### Promise Checks
- [ ] The promise is specific (not "grow your business")
- [ ] The promise is believable given the proof
- [ ] The promise doesn't require the reader to already trust you

### Scoring
```
Stranger Test: X/13 checks passing
- If < 10: offer needs clarity rework
- If 10-12: minor polish needed
- If 13: pass
```

---

## Audit Pass 2: The Competitor Test (Step 11)

**Question:** "If a competitor offered the exact same thing, would yours still win?"

### Differentiation Checks
- [ ] Named mechanism or unique method (not generic "coaching" or "consulting")
- [ ] At least one thing a competitor can't easily copy
- [ ] Identity/story element that's genuinely yours
- [ ] Proof that's specific to you (not generic industry stats)

### Positioning Checks
- [ ] Clear category or category creation ("I'm not a [generic], I'm a [specific]")
- [ ] Price justified by value, not just "competitive"
- [ ] Guarantee that competitors aren't offering
- [ ] Specificity level higher than alternatives

### "So What" Chain
For every claim in the offer, ask "so what?" until you hit an emotional truth:
- Feature → Benefit → Emotional benefit → Identity benefit
- If the chain breaks before reaching emotion, the claim needs strengthening

### Scoring
```
Competitor Test: X/8 checks passing
- If < 5: offer is too generic — needs unique mechanism work
- If 5-7: competitive but could be sharper
- If 8: strong differentiation
```

## Audit Pass 3: The Objection Test (Step 12)

**Question:** "What would make a qualified buyer say no, and have we addressed it?"

### Common Objections Checklist
For each, check: is there a counter in the offer document?

| Objection | Addressed? | Where/How |
|-----------|-----------|-----------|
| "Too expensive" | [ ] | [price anchor, ROI framing, guarantee] |
| "I've tried something like this" | [ ] | [unique mechanism, why this is different] |
| "I don't have time" | [ ] | [timeline, done-for-you elements, simplicity] |
| "I'm not sure it'll work for ME" | [ ] | [specific proof for their situation, guarantee] |
| "I need to think about it" | [ ] | [urgency, scarcity, risk reversal] |
| "I don't know who you are" | [ ] | [authority proof, demonstrations, free value] |
| "I can do this myself" | [ ] | [time cost, expertise gap, hidden complexity] |
| "What if it doesn't work?" | [ ] | [guarantee terms, risk reversal] |

### Trust Gap Analysis
- [ ] Every claim has supporting evidence (not just assertions)
- [ ] Guarantee specifically addresses the biggest fear
- [ ] At least 2 trust signals scoring 2+ (from carrier-trust-signals.md)
- [ ] Price is anchored to value, not just stated

### Scoring
```
Objection Test: X/12 checks passing
- If < 8: major trust gaps — address objections before deploying
- If 8-11: solid but patch remaining gaps
- If 12: comprehensive objection handling
```

## When to Re-Audit

- After any significant offer change (price, promise, mechanism)
- Before launching a new campaign based on this offer
- Quarterly as a hygiene check
- When conversion rates drop unexpectedly
